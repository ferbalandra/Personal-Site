Scenario-Driven Design Systems

Yesenia Perez-Cruz, Design Director, Vox Media

Design is scaling. Increasingly complex design problems. Distributed teams.

When the American people go online to access government services, they’re often met with confusing navigation systems, an array of visual brands, and inconsistent interaction patterns. Creating consistency between these systems will help people more efectively access the services they need an increase their trust in the government.

―Maya Benari

When design is consistent it gives the user more of a safe feeling chesive, unified

Design System

A system is an interconnected set of elements coherently organized in a way that achieves something. Systems can fail if they dont have a standard purpose and if they are difficult to implement.

Start your design system with user-scenarios.

Why

Easier to design, build, launch, maintain, and evolve the sites on platforms. Tell better stories faster

Needed to support different designs but unify into one code system, but allow brands to feel distinct

Flexible components Issues

When using modular systems for several sites they look consistent but they loose personality and we do not take the user into account.

  1. Sites felt too similar
  2. Did not reflect critical diferences in content, tone, and audience

Modules didn’t have a clear purpose. And sites may loose personality

Successful design systems start with content and people.

Implement it better

  1. Start with fast, unified platform
  2. Be scenario-driven when creating variations
  3. Find key moments for visual brand expression that serve our audience

Do a UI inventory of the design system or elements used. Group items into purpose and mapped them in a user story. Why or how user use these UI elements and how

Keeping this map in my mind helped me think in families of patterns joined by a shared purpose, rather than individual pages.

By doing this we can identify elements and design better components that have the information and fucntion that is actually needed

  • Identify core workflows and the patterns that need to support these workflows.
  • Understand the role each pattern plays in a user’s journey.
  • Define how the patterns work together to create a cohesive experience

Identifying Core Workflows

Audience goals consistent across brands, but content difered. Only add a layout if there’s a content need.

Content Audit is necessary to look at the elements which can be included in the new system. Does this component add value ? Is it available to more than 3 brands ? is it esential ?

Name components collaboratively, The purpose of each component into the system if you collaborate with names it becomes easier to keep it consistent

Flexibility levers Brad Frost talks about elements that can be different but still keeps the look consistent Pattern Variations
